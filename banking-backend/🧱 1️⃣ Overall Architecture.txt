📁 PACKAGE STRUCTURE — What & Why
🔹 entity
Example:

Customer

Transaction

Why?

Represents database tables using JPA.

Annotations Used:
@Entity
@Table(name = "customers")

Meaning:

This class maps to DB table.

@Id
@GeneratedValue(strategy = GenerationType.IDENTITY)

Meaning:

Primary key auto increment.

@Column(nullable = false)

Meaning:

DB column constraints.

🔹 repository

Example:

CustomerRepository

TransactionRepository

public interface CustomerRepository extends JpaRepository<Customer, Long>

Why?

JPA gives built-in CRUD

No need to write SQL manually

Custom methods like:

Optional<Customer> findByEmail(String email);
List<Customer> findByStatus(String status);
Page<Customer> findAll(Pageable pageable);

Spring auto-generates queries from method names.

🔹 dto (VERY IMPORTANT)

You created:

RegisterRequest

LoginRequest

CustomerResponse

AdminUserResponse

ErrorResponse

Why DTO?

To:

Hide entity fields (like password)

Control API response

Avoid exposing DB structure

Improve security

Never return entity directly in production systems.

🔹 service

Business logic layer.

Example:

AuthService

CustomerService

AdminService

TransactionService

Why Service Layer?

Controllers should NOT:

Access repository directly

Contain business logic

Service:

Validates

Applies rules

Controls access logic

Performs operations

🔹 service.impl

Actual implementations.

Example:

@Service
public class AuthServiceImpl implements AuthService
Why @Service?

Tells Spring:

This is a business component → manage it as a bean.

Important config:

Meaning:
Only admin can access those endpoints.

🔹 JWT Flow
Login Flow

User sends email/password

AuthenticationManager authenticates

If valid → JWT token generated

Token returned to frontend

🔹 JwtAuthenticationFilter

Purpose:

Intercepts every request

Extracts token

Validates token

Sets authentication in SecurityContext

Without this, JWT won’t work.
